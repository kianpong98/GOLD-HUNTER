Gold Hunter News Engine v10.2.3 — KV Write Protection

Changes are backend-only. Website UI, SEO, layouts and market tools are unchanged.

- Official BLS/FRED/Federal Reserve caches write only when meaningful data changes.
- Admin Forecast/Event save skips KV.put when submitted values are unchanged.
- Archived event data writes only when the stored data differs.
- ETF cache no longer rewrites identical static data on every request.
- Quotes cache is throttled to reduce shared KV daily writes.
- High-frequency release checks may continue reading official sources without consuming a KV write on every check.

Important: an already exhausted Cloudflare daily KV quota must reset before a new write can succeed.
