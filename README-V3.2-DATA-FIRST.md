# Gold Hunter V3.2 — Data First

This version separates the data jobs into three verified GitHub Actions workflows:

- `Update official calendar data`
  - BLS: CPI, Core CPI, PPI, Core PPI, NFP, Unemployment Rate
  - FRED official-source series: Retail Sales, Initial Jobless Claims, GDP, PCE, Core PCE and FOMC target range
  - Writes `data/official-data.json`

- `Update SPDR GLD holdings`
  - Downloads the official SPDR GLD historical archive
  - Writes the latest 30 records to `assets/data/spdr-gld-holdings.json`

- `Update quote cache`
  - Stores delayed last-known XAU and DXY prices in `data/quotes.json`
  - Used as a fallback when live quote requests fail

## Important GitHub note

Scheduled workflows only run from the repository's default branch. Test the workflows on the feature branch through their push triggers, then merge the workflow files to `main` for the schedules to continue automatically.

## Cloudflare

Keep these bindings:

- KV binding: `GH_MARKET_DATA`
- Secret: `ADMIN_PIN`

The calendar endpoint now prefers the verified static cache generated by GitHub Actions, while keeping runtime BLS/FRED fetches as fallback.
